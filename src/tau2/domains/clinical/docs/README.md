# Clinical Domain Documentation

This directory contains comprehensive documentation for all clinical domains in the tau2 benchmark.

## Structure

Each clinical specialty has its own documentation subdirectory:

- **[cardiology/](cardiology/)** - Cardiovascular medicine documentation
  - Cardiology tools definition
  - Cardiology clinical policy

- **[endocrinology/](endocrinology/)** - Endocrine system documentation
  - Endocrinology tools definition
  - Endocrinology clinical policy

- **[gastroenterology/](gastroenterology/)** - Digestive system documentation
  - Gastroenterology tools definition
  - Gastroenterology clinical policy

- **[nephrology/](nephrology/)** - Kidney and renal system documentation
  - Nephrology tools definition
  - Nephrology clinical policy

- **[neurology/](neurology/)** - Nervous system documentation
  - Neurology tools definition
  - Neurology clinical policy

## Documentation Files

### Tools Definition Files
Each domain has a `*_TOOLS_DEFINITION.md` file that contains:
- Patient information tools
- Condition-specific assessment tools
- Additional recommended tools
- Clinical workflow integration
- References and review checklist

### Policy Files
Each domain has a `*_POLICY.md` file that contains:
- Clinical philosophy and scope
- Patient evaluation protocols
- Clinical protocols
- Safety protocols
- Quality assurance
- References and approvals

### Medical Review Summary
The `MEDICAL_REVIEW_SUMMARY.md` file provides:
- Overall review process
- Document structure overview
- Key areas requiring expert validation
- Review timeline and phases
- Success criteria

## Important Notice

⚠️ **All clinical documentation requires validation by board-certified medical specialists**

These documents are for medical expert review only and must not be used for patient care until approved.
