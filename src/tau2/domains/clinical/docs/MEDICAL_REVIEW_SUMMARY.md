## 📁 Document Structure

### Tools Definition Documents (`*_TOOLS_DEFINITION.md`)

Each department has a comprehensive tools definition document containing:

1. **Patient Information Tools**
   - Patient retrieval by MRN
   - Demographic and clinical data access

2. **Condition-Specific Assessment Tools**
   - Cardiology: BP assessment, QTc calculation, heart rate interpretation
   - Neurology: Stroke risk, headache classification, seizure evaluation
   - Nephrology: eGFR calculation, kidney function assessment, medication dosing
   - Gastroenterology: LFT interpretation, liver fibrosis, anemia evaluation
   - Endocrinology: Glucose interpretation, HbA1c interpretation, thyroid function

3. **Additional Recommended Tools** (Pending Implementation)
   - Each department lists 3-7 additional tools for future development

4. **Clinical Workflow Integration**
   - Recommended tool usage sequences
   - Decision support algorithms

5. **References and Review Checklist**
   - Current guideline citations
   - Reviewer sign-off sections

### Policy Documents (`*_POLICY.md`)

Each department has a comprehensive policy document containing:

1. **Clinical Philosophy & Scope**
   - Mission statement
   - Scope of practice (in-scope and out-of-scope)
   - Decision support principles

2. **Patient Evaluation Protocols**
   - Initial consultation workflows
   - Mandatory documentation requirements

3. **Clinical Protocols**
   - 2-4 major condition protocols per department
   - Red flag identification
   - Management algorithms

4. **Safety Protocols**
   - Emergency identification
   - Tool limitations and fail-safes

5. **Quality Assurance**
   - Performance monitoring
   - Clinical decision review

6. **References and Approvals**
   - Current guideline citations
   - Required review signatures

### Phase 3: Revision (Week 7-8)

- [ ] Prioritize changes (critical, high, medium, low)

- [ ] Document rationale for any rejected recommendations
- [ ] Prepare revised documents

### Phase 4: Secondary Review (Week 9-10)
- [ ] Distribute revised documents to reviewers

- [ ] Document any remaining concerns

### Phase 5: Approval & Implementation (Week 11-12)
- [ ] Final approval from clinical governance
- [ ] Update production tools and policies

- [ ] Schedule next review cycle (annually)
