# 数据质量测试模块架构

## 负责模块

**数据质量测试由 `DataQualityFiltering/data_quality/` 目录负责**

---

## 模块结构

```
DataQualityFiltering/
├── data_quality/                          ← 数据质量测试主目录
│   ├── validators/                        ← 验证器（不需要LLM）
│   │   └── medical_dialogue_validator.py  医学对话验证器
│   ├── filter_engine.py                   ← 过滤引擎（不需要LLM）
│   ├── llm_review.py                      ← LLM审查器（需要LLM API）
│   ├── human_review.py                    ← 人工审查
│   ├── pipeline.py                        ← 完整管道
│   └── check_medical_dialogue.py          ← CLI工具
│
├── modules/                               ← 我们刚实现的模块
│   ├── scenario_classifier.py             ← 场景分类器
│   ├── inquiry_threshold_validator.py     ← 追问阈值验证器
│   └── safety_validator.py                ← 安全验证器
│
└── [根目录]
    ├── scoring.py                         ← 评分系统
    ├── quality_threshold_pipeline.py      ← 质量阈值管道
    └── tasks_json_improver_v2.py          ← 任务改进器V2
```

---

## 各模块职责

### 1️⃣ **基础验证层**（不需要LLM）

| 模块 | 文件 | 功能 | 命令 |
|------|------|------|------|
| **医学对话验证器** | `validators/medical_dialogue_validator.py` | 检查数据格式、字段完整性、医学关键词 | `python -m DataQualityFiltering data-quality check` |
| **场景分类器** | `modules/scenario_classifier.py` | 自动识别场景类型（6种） | 内部调用 |
| **追问阈值验证器** | `modules/inquiry_threshold_validator.py` | 生成追问阈值规则 | 内部调用 |
| **安全验证器** | `modules/safety_validator.py` | 生成安全检查规则 | 内部调用 |

### 2️⃣ **质量评分层**（需要LLM）

| 模块 | 文件 | 功能 | API要求 |
|------|------|------|---------|
| **LLM审查器** | `data_quality/llm_review.py` | 用LLM给任务打分（4个维度） | ✅ 需要LLM API |
| **人工审查器** | `data_quality/human_review.py` | 人工审查界面 | ❌ 不需要API |
| **评分系统** | `scoring.py` | 0-30分制评分计算 | ❌ 不需要API |

### 3️⃣ **处理层**（不需要LLM）

| 模块 | 文件 | 功能 | 命令 |
|------|------|------|------|
| **过滤引擎** | `data_quality/filter_engine.py` | 根据分数过滤任务 | `python -m DataQualityFiltering data-quality filter` |
| **质量阈值管道** | `quality_threshold_pipeline.py` | 三级分类（高/中/低） | `python -m DataQualityFiltering data-quality quality-threshold` |
| **任务改进器** | `tasks_json_improver_v2.py` | 数据增强（添加规则） | `python DataQualityFiltering/tasks_json_improver_v2.py` |

## 模块调用关系

```
用户命令
   ↓
┌──────────────────────────────────────────┐
│  CLI 入口 (__main__.py)                  │
└──────────────────────────────────────────┘
   ↓
   ├─ data-quality check
   │   └─→ MedicalDialogueValidator
   │       ├─ 检查JSON格式 ✓
   │       ├─ 检查字段完整性 ✓
   │       └─ 检查医学关键词 ✓
   │
   ├─ data-quality filter
   │   └─→ QualityFilter
   │       ├─ 读取分数
   │       ├─ 按阈值过滤
   │       └─ 输出结果
   │
   ├─ data-quality quality-threshold
   │   └─→ QualityThresholdPipeline
   │       ├─ 读取任务
   │       ├─ 读取分数（或用默认）
   │       ├─ 三级分类
   │       └─ 保存结果
   │
   └─ tasks_json_improver_v2.py
       └─→ TasksJsonImproverV2
           ├─ ScenarioClassifier
           ├─ InquiryThresholdValidator  ← 我们刚实现的
           ├─ SafetyValidator
           ├─ UncertaintyHandler
           └─ ComplexityEnhancer
```

## 哪些模块需要LLM API？

| 模块 | 需要API？ | 原因 |
|------|----------|------|
| `MedicalDialogueValidator` | ❌ 不需要 | 规则验证（格式、字段） |
| `ScenarioClassifier` | ❌ 不需要 | 关键词匹配 |
| `InquiryThresholdValidator` | ❌ 不需要 | 配置生成 |
| `SafetyValidator` | ❌ 不需要 | 规则生成 |
| `QualityFilter` | ❌ 不需要 | 分数过滤 |
| `QualityThresholdPipeline` | ❌ 不需要 | 分级逻辑 |
| `TasksJsonImproverV2` | ❌ 不需要 | 元数据生成 |
| `LLMReviewer` | ✅ **需要** | LLM评分 |

## 完整的数据质量测试流程

### 无API模式（当前能做的）

```bash
# 步骤1: 格式验证
python -m DataQualityFiltering data-quality check --input tasks.json
→ ✓ 格式正确
→ ✓ 字段完整

# 步骤2: 数据增强
python DataQualityFiltering/tasks_json_improver_v2.py --output improved.json
→ ✓ 添加场景分类
→ ✓ 添加追问阈值规则
→ ✓ 添加安全规则

# 步骤3: 质量筛选（使用默认分数）
python -m DataQualityFiltering data-quality quality-threshold \
  --input improved.json --no-improvement
→ ✓ 三级分类（但分数是默认值）
```

### 有API模式（完整功能）

```bash
# 步骤1: 格式验证
python -m DataQualityFiltering data-quality check --input tasks.json

# 步骤2: LLM评分（需要API）
export OPENAI_API_KEY="sk-..."
python -m DataQualityFiltering data-quality llm-review \
  --input tasks.json --output scores.json

# 步骤3: 质量筛选（使用真实分数）
python -m DataQualityFiltering data-quality quality-threshold \
  --input tasks.json --scores-file scores.json
→ ✓ 三级分类（使用真实分数）
```

## 总结

**数据质量测试由两个层次的模块负责**：

1. **基础层**（`data_quality/` 目录）
   - 不需要LLM
   - 做格式验证、规则生成、数据增强

2. **评分层**（`llm_review.py` + `scoring.py`）
   - `LLMReviewer` 需要LLM API
   - `ScoringSystem` 不需要API（只是计算）

**当前状态**：
- ✅ 基础层完全可用
- ❌ 评分层需要API才能获得真实分数

**测试瓶颈**：
- 不是系统设计问题
- 是缺少LLM API调用
- 解决方案：提供API Key或人工预先评分
