# DataQualityFiltering 测评指南

## 快速开始

### 1. 无需 LLM API 的基础测评

#### 测试任务改进系统（V2.0 + 追问阈值）
```bash
# 测试单个任务
python DataQualityFiltering/tasks_json_improver_v2.py \
  --limit 1 \
  --output test_output.json

# 查看结果中的 inquiry_threshold_validation 字段
cat test_output.json | jq '.[0].inquiry_threshold_validation'
```

**预期输出**：
```json
{
  "scenario_type": "MEDICATION_CONSULTATION",
  "risk_level": "HIGH",
  "risk_emoji": "🔴",
  "threshold_config": {
    "min_questions_before_advice": 4,
    "allowed_response_type": "SAFETY_WARNING_ONLY"
  },
  "demographic_requirements": [...],
  "clinical_requirements": [...]
}
```

#### 测试质量阈值筛选
```bash
python -m DataQualityFiltering data-quality quality-threshold \
  --input chinese_internal_medicine_tasks_improved.json \
  --output-dir ./test_results \
  --no-improvement
```

**无需 API Key**，直接运行即可看到三级分类结果。

---

### 2. 需要 LLM API 的高级测评

#### 安装依赖
```bash
pip install openai anthropic
```

#### 设置 API Key
```bash
# OpenAI
export OPENAI_API_KEY="sk-..."

# 或 Anthropic
export ANTHROPIC_API_KEY="sk-ant-..."
```

#### 运行 LLM 评估
```bash
python -m DataQualityFiltering agent-eval evaluate \
  --input task.json \
  --model gpt-4-turbo \
  --output result.json
```

## 输出说明

### 无 LLM 模式输出
- 场景分类（INFORMATION_QUERY, MEDICATION_CONSULTATION 等）
- 风险等级（🔴 HIGH / 🟡 MEDIUM / 🟢 LOW）
- 追问阈值配置
- 安全验证规则

### LLM 评估模式输出
- AI 回答质量评分
- 多维度评估（临床准确性、安全性等）
- 详细改进建议
