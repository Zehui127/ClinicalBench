# 数据测试模块完整架构

## 问题：数据是什么模块测试的？

**答案**：数据测试分为**三个层次**，由不同的模块负责：

```
┌─────────────────────────────────────────────────────────────┐
│  第1层: 数据质量测试（测试"任务"本身）                       │
│  模块: DataQualityFiltering/data_quality/                   │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  第2层: 数据增强验证（测试"规则生成"）                       │
│  模块: DataQualityFiltering/modules/                        │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  第3层: Agent表现测试（测试"AI回答"）                        │
│  模块: src/tau2/evaluator/                                  │
└─────────────────────────────────────────────────────────────┘
```

---

## 第1层：数据质量测试

**目标**：测试任务（tasks）本身的质量

**模块位置**：`DataQualityFiltering/data_quality/`

### 子模块

| 模块 | 文件 | 测试内容 | 需要API |
|------|------|---------|---------|
| **格式验证器** | `validators/medical_dialogue_validator.py` | JSON格式、字段完整性、医学关键词 | ❌ 不需要 |
| **LLM审查器** | `llm_review.py` | 4个维度评分（0-5分） | ✅ 需要 |
| **过滤引擎** | `filter_engine.py` | 根据分数过滤 | ❌ 不需要 |
| **质量阈值管道** | `quality_threshold_pipeline.py` | 三级分类（高/中/低） | ❌ 不需要 |

### 测试什么？

```
输入: tasks.json
{
  "id": "task_001",
  "ticket": "高血压能吃党参吗？",
  "user_scenario": {...},
  "evaluation_criteria": {...}
}

↓ 测试

输出1: 格式验证
  ✓ JSON格式正确
  ✓ 包含必需字段
  ✓ 有医学关键词

输出2: 质量评分
  - 临床准确性: 4.5/5
  - 场景真实性: 4.0/5
  - 评估完整性: 3.5/5
  - 难度适当性: 4.0/5

输出3: 质量分级
  - 高质量（≥27分）
  - 中等质量（24-26分）
  - 低质量（<24分）
```

---

## 第2层：数据增强验证

**目标**：测试生成的规则（场景分类、追问阈值、安全规则）

**模块位置**：`DataQualityFiltering/modules/`

### 子模块

| 模块 | 文件 | 测试/生成内容 | 需要API |
|------|------|-------------|---------|
| **场景分类器** | `scenario_classifier.py` | 识别6种场景类型 | ❌ 不需要 |
| **追问阈值验证器** | `inquiry_threshold_validator.py` | 生成追问阈值规则 | ❌ 不需要 |
| **安全验证器** | `safety_validator.py` | 生成安全检查规则 | ❌ 不需要 |

### 测试/生成什么？

```
输入: tasks.json
{
  "ticket": "能吃硝苯地平吗？",
  ...
}

↓ 测试/生成

输出: 场景分类
  - scenario_type: "MEDICATION_CONSULTATION"
  - risk_level: "HIGH"
  - 追问阈值规则:
    - 必须问: 年龄、药名、剂量、其他用药
    - 最小问题数: 4
    - 响应类型: 只给安全警告+就医建议
```

---

## 第3层：Agent表现测试

**目标**：测试AI Agent在实际任务中的表现

**模块位置**：`src/tau2/evaluator/`

### 子模块

| 模块 | 文件 | 测试内容 | 需要API |
|------|------|---------|---------|
| **临床准确性评估器** | `evaluator_clinical_accuracy.py` | 医学知识、诊断、治疗 | ✅ 需要 |
| **对话流畅性评估器** | `evaluator_dialogue_fluency.py` | 理解、连贯、清晰 | ✅ 需要 |
| **安全同理心评估器** | `evaluator_safety_empathy.py` | 安全意识、同理心 | ✅ 需要 |
| **综合医疗评估器** | `evaluator_clinical.py` | 整合三个维度 | ✅ 需要 |

### 测试什么？

```
输入:
  - 任务: "患者问能吃硝苯地平吗？"
  - AI对话: "您好，请问您今年多大了？..."

↓ 测试

输出: AI表现评分
  - 临床准确性: 4.2/5
    - ✓ 询问了年龄
    - ✓ 询问了药名
    - ✓ 询问了其他用药
    - ✓ 包含安全警告

  - 对话流畅性: 4.5/5
    - ✓ 问题理解正确
    - ✓ 回答连贯
    - ✓ 表达清晰

  - 安全同理心: 4.0/5
    - ✓ 有安全意识
    - ✓ 有同理心表达

总分: 4.2/5 (通过)
```

---

## 三层测试对比

| 层次 | 测试对象 | 测试内容 | 主要模块 | 需要API |
|------|---------|---------|---------|---------|
| **第1层** | 任务本身 | 格式、质量分数 | `data_quality/` | LLM评分需要 |
| **第2层** | 规则生成 | 场景分类、阈值规则 | `modules/` | ❌ 不需要 |
| **第3层** | AI回答 | 临床准确性、对话、安全 | `tau2/evaluator/` | ✅ 需要 |

```
      ↓
┌─────────────────────────────────┐

└─────────────────────────────────┘
      ↓
┌─────────────────────────────────┐
│  第2层: 数据增强                  │
│  - 场景分类                       │

│  - 安全验证规则                   │

      ↓

```

## 命令对照

### 第1层：数据质量测试

```bash
# 格式验证
python -m DataQualityFiltering data-quality check \
  --input tasks.json

# 质量评分（需要API）
python -m DataQualityFiltering data-quality llm-review \
  --input tasks.json \
  --model gpt-4o-mini

# 质量过滤
python -m DataQualityFiltering data-quality filter \
  --input tasks.json \
  --threshold 3.5

# 质量阈值筛选
python -m DataQualityFiltering data-quality quality-threshold \
  --input tasks.json
```

### 第2层：数据增强

```bash
# 任务改进（V2.0）
python DataQualityFiltering/tasks_json_improver_v2.py \
  --input tasks.json \
  --output improved_tasks.json

# 输出包含:
# - 场景分类
# - 追问阈值规则
# - 安全验证规则
```

### 第3层：Agent测试（在tau2框架中）

```python
# 在tau2框架中运行
from tau2.evaluator.evaluator_clinical import ClinicalEvaluator

# 评估AI表现
reward = ClinicalEvaluator.calculate_reward(
    task=task,
    full_trajectory=messages,  # AI对话历史
    model="gpt-4"
)
```

## 总结

**"数据是什么模块测试的？"**

**答案**：数据测试分三层：

1. **第1层 - 数据质量测试**
   - 模块：`DataQualityFiltering/data_quality/`
   - 测试：任务格式、质量分数
   - 状态：✅ 部分可用（LLM评分需要API）

2. **第2层 - 规则生成验证**
   - 模块：`DataQualityFiltering/modules/`
   - 测试：场景分类、追问阈值
   - 状态：✅ 完全可用

3. **第3层 - Agent表现测试**
   - 模块：`src/tau2/evaluator/`
   - 测试：AI的实际回答质量
   - 状态：❌ 需要API + AI对话数据

**关键区别**：
- 第1层测试：**"任务本身好不好？"**
- 第2层测试：**"规则生成对不对？"**
- 第3层测试：**"AI回答好不好？"**
