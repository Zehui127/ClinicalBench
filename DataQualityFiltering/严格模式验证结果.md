# 严格验证结果分析

## 问题发现

虽然我们设置了更高的阈值（`--min-patient-length 20`），但结果还是一样的（95.8%通过率）。

**原因**：阈值参数只产生**警告**，不会导致**失败**。

---

## 当前验证逻辑

### 导致失败的条件（issues）
- ❌ 缺少dialogue_id
- ❌ 缺少department信息
- ❌ 对话轮数不足（< 2）
- ❌ 未检测到医学关键词

### 只产生警告的条件（warnings）
- ⚠️ 患者主诉过短（< min_patient_length）
- ⚠️ 医生回答过短（< min_doctor_length）
- ⚠️ 医学关键词较少（< 2）

**关键**：即使主诉只有1个字符，只要有医生回答和医学关键词，就会**通过**！

---

## 严格验证结果（当前）

```
总对话数: 220,606
通过: 211,400 (95.8%)
失败: 9,206 (4.2%)
```

**警告分布**：
- 患者主诉过短（<20字符）: 38,487条
- 医学回答过短（<30字符）: 735条
- 医学关键词较少（<2个）: 20,601条

**虽然有警告，但这些对话仍然通过了！**

---

## 如何实现真正的严格验证？

### 方法1：使用 `--strict` 模式

```bash
python -m DataQualityFiltering raw-data validate \
  --input "./data/raw/medical_dialogues/chinese_meddialog/Data_数据/IM_内科/内科5000-33000.csv" \
  --output validation_very_strict.json \
  --encoding gbk \
  --strict
```

**效果**：警告也会导致失败

### 方法2：过滤掉有警告的对话

通过分析详细报告，可以过滤掉：
- 主诉 < 20字符的对话
- 医学关键词 < 2的对话

---

## 需要我运行严格模式吗？

使用 `--strict` 模式会把通过率从95.8%大幅降低（可能降到60-70%），但数据质量会更高。
