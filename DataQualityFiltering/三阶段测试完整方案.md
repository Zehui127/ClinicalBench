## 阶段1：原始数据验证器

### 功能

验证原始医患对话数据是否适合转换为tasks

### 使用方法

```bash
python -m DataQualityFiltering raw-data validate \
  --input raw_dialogues.json \
  --output validation_report.json
```

### 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--input` | 输入文件路径（CSV或JSON） | 必需 |
| `--output` | 输出报告路径 | 可选 |
| `--format` | 文件格式（csv/json/auto） | auto |
| `--encoding` | 文件编码 | utf-8 |
| `--strict` | 严格模式 | - |
| `--min-patient-length` | 患者主诉最小长度 | 5 |
| `--min-doctor-length` | 医生回答最小长度 | 10 |
| `--min-turns` | 最少对话轮数 | 2 |

### 验证项

| 检查项 | 说明 | 失败条件 |
|--------|------|---------|
| 必需字段 | dialogue_id, department | 缺失 |
| 对话轮数 | 至少N轮对话 | < min_turns |
| 患者主诉 | 必须有患者发言 | 无患者轮次 |
| 医生回答 | 建议有医生发言 | 无医生轮次（警告） |
| 医学关键词 | 必须包含医学词汇 | 无关键词 |
| 内容长度 | 内容不能太短 | 过短（警告） |

### 示例输出

```

  - 对话轮数不足：1 < 2: 3
  - 未检测到医学关键词，可能不是医学咨询: 3

主要警告:
  - 缺少医生回答: 2
  - 患者主诉过短：1 < 5: 1
  - 患者主诉过短：2 < 5: 1

============================================================
```

## 完整工作流程

### 推荐流程

```bash
# 1. 验证原始数据
python -m DataQualityFiltering raw-data validate \
  --input raw_dialogues.json \
  --output validation_report.json

# 2. 转换数据（通过率达标后）
python convert_chinese_meddialog.py \
  --input raw_dialogues.json \
  --output tasks.json

# 3. 数据增强
python DataQualityFiltering/tasks_json_improver_v2.py \
  --input tasks.json \
  --output enhanced_tasks.json

# 4. 质量评分（可选，需要API）
python -m DataQualityFiltering data-quality llm-review \
  --input enhanced_tasks.json \
  --output scores.json

# 5. 质量筛选
python -m DataQualityFiltering data-quality quality-threshold \
  --input enhanced_tasks.json \
  --scores-file scores.json
```
