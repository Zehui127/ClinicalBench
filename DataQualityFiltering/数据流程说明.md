# DataQualityFiltering 处理的数据类型

## 核心答案

**`DataQualityFiltering/data_quality/` 处理的是 → 已经生成的 `tasks.json`，不是原始医患对话**

---

## 完整数据流程

```
┌──────────────────────────────────────────────────────────┐
│  阶段1: 原始医患对话（Raw Dialogues）                      │
│  来源: Chinese MedDialog Dataset                          │
│  格式: CSV文件，包含医患对话文本                           │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│  阶段2: 数据转换（Conversion）                             │
│  脚本: convert_chinese_meddialog.py                       │
│  功能: 将原始对话转换为tau2-bench格式的tasks              │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│  阶段3: 生成的Tasks（Generated Tasks）                    │
│  文件: chinese_internal_medicine_tasks_original.json     │
│  格式: tau2-bench标准格式                                 │
│         (id, ticket, user_scenario, evaluation_criteria)  │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│  阶段4: 数据质量测试（Data Quality Testing） ← 我们在这里！│
│  模块: DataQualityFiltering/data_quality/                 │
│  功能: 验证、评分、过滤tasks                              │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│  阶段5: 数据增强（Data Enhancement）                       │
│  脚本: tasks_json_improver_v2.py                          │
│  功能: 添加场景分类、追问阈值、安全规则                    │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│  阶段6: 最终Tasks（Final Tasks）                          │
│  文件: chinese_internal_medicine_tasks_v2.json           │
│  用途: 用于Agent评估和测试                                │
└──────────────────────────────────────────────────────────┘
```

## 各阶段数据示例

### 阶段1: 原始医患对话（CSV格式）

```csv
department,dialogue_id,role,content
内科,D001,患者,高血压患者能吃党参吗？
内科,D001,医生,高血压患者可以适量服用党参...
```

**特点**：
- 纯文本对话
- 无结构化字段
- 需要转换才能使用

---

### 阶段3: 生成的Tasks（JSON格式）

```json
{
  "id": "clinical_internal_medicine_0",
  "ticket": "我有高血压这两天女婿来的时候给我拿了些党参泡水喝，您好高血压可以吃党参吗？",
  "user_scenario": {
    "instructions": {
      "domain": "internal_medicine",
      "reason_for_call": "高血压患者能吃党参吗？",
      "known_info": "我有高血压..."
    }
  },
  "evaluation_criteria": {
    "actions": [
      {"action_id": "provide_medical_advice"}
    ]
  }
}
```

**特点**：
- ✅ 结构化格式
- ✅ 包含必需字段
- ✅ 可以直接用于测试

**这是 `data_quality/` 处理的数据格式！**

---

### 阶段6: 增强后的Tasks（V2.0）

```json
{
  "id": "clinical_internal_medicine_0",
  "ticket": "我有高血压...",
  "user_scenario": {...},
  "evaluation_criteria": {...},
  "scenario_type": "INFORMATION_QUERY",
  "inquiry_threshold_validation": {
    "scenario_type": "INFORMATION_QUERY",
    "risk_level": "LOW",
    "min_questions_before_advice": 0
  },
  "safety_validation": {...},
  "quality_metadata_v2": {...}
}
```

**特点**：
- ✅ 包含场景分类
- ✅ 包含追问阈值规则
- ✅ 包含安全验证规则
- ✅ 包含质量元数据

## data_quality/ 模块处理什么？

### ✅ 处理的内容

| 模块 | 输入 | 处理 | 输出 |
|------|------|------|------|
| **MedicalDialogueValidator** | tasks.json | 检查格式、字段 | 验证报告 |
| **LLMReviewer** | tasks.json | LLM打分（0-5分） | 分数列表 |
| **QualityFilter** | tasks.json + 分数 | 按阈值过滤 | 过滤后的tasks |

### ❌ 不处理的内容

| 内容 | 原因 |
|------|------|
| 原始医患对话（CSV） | 不属于职责范围 |
| 数据转换（CSV→JSON） | 由 `convert_chinese_meddialog.py` 负责 |
| Agent实际回答 | 由 `agent_evaluation/` 负责 |

## 命令对照

### 数据转换（不是 data_quality/ 负责）

```bash
# 这是数据转换脚本
python convert_chinese_meddialog.py
→ 输入: 原始CSV文件
→ 输出: chinese_internal_medicine_tasks_original.json
```

### 数据质量测试（data_quality/ 负责）

```bash
# 格式验证
python -m DataQualityFiltering data-quality check \
  --input chinese_internal_medicine_tasks_original.json
→ 输入: tasks.json
→ 输出: 验证报告

# 质量评分
python -m DataQualityFiltering data-quality llm-review \
  --input chinese_internal_medicine_tasks_original.json
→ 输入: tasks.json
→ 输出: 分数列表

# 质量过滤
python -m DataQualityFiltering data-quality filter \
  --input chinese_internal_medicine_tasks_original.json
→ 输入: tasks.json + 分数
→ 输出: 过滤后的tasks.json
```

### 数据增强（也不是 data_quality/ 负责）

```bash
# 任务改进系统（我们刚实现的）
python DataQualityFiltering/tasks_json_improver_v2.py \
  --input chinese_internal_medicine_tasks_improved.json
→ 输入: tasks.json
→ 输出: 增强后的tasks.json（带追问阈值规则）
```

## 总结

### 数据流程

```
原始对话（CSV）
    ↓
[数据转换脚本] ← 不属于data_quality/
    ↓
生成的tasks（JSON） ← data_quality/ 的输入
    ↓
[data_quality/ 验证/评分/过滤] ← data_quality/ 的职责
    ↓
增强后的tasks（JSON）
```

### data_quality/ 的定位

**`data_quality/` 是质检部门**：
- ✅ 检查产品质量（tasks格式是否正确）
- ✅ 给产品打分（tasks质量如何）
- ✅ 过滤次品（移除低质量tasks）

**不是生产部门**：
- ❌ 不负责从原材料生产产品（CSV→JSON转换）
- ❌ 不负责产品升级（添加追问阈值规则）

---

## 模块职责划分

| 模块 | 职责 | 处理的数据 |
|------|------|-----------|
| `convert_chinese_meddialog.py` | 数据转换 | 原始对话 → tasks |
| `data_quality/` | 质量控制 | tasks → 验证/评分/过滤 |
| `tasks_json_improver_v2.py` | 数据增强 | tasks → 增强tasks |
| `agent_evaluation/` | Agent测试 | Agent回答 → 评估 |

---

## 回答您的问题

**DataQualityFiltering/data_quality/ 是对进来的原始数据，还是对原始数据生成后的task？**

**答案**：对**原始数据生成后的task**（tasks.json格式）

具体来说：
1. 原始数据 = 医患对话CSV文件
2. 生成的task = tasks.json文件
3. **data_quality/ 处理 = tasks.json文件** ← 这里
