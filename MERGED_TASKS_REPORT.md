## 追问阈值规则更新

### 合并任务的规则增强
所有合并任务的追问阈值规则都得到增强：

1. **风险等级**: 选择合并任务中的最高风险
2. **最小问题数**: 原始阈值 + 1
   - 例: 原始MEDIUM(3问题) → 合并后MEDIUM(4问题)
3. **标记**: 添加 `merged_from_multiple: true`
4. **来源记录**: 保存所有原始任务ID

### 合并规则优先级
```
MEDICATION_CONSULTATION (最高)
  ↓
SYMPTOM_ANALYSIS
  ↓
CHRONIC_MANAGEMENT
  ↓
EMERGENCY_CONCERN
  ↓
LIFESTYLE_ADVICE
  ↓
INFORMATION_QUERY (最低)
```

## 技术实现

### 相似度计算
```python
相似度 = 场景类型相似度 × 0.3
       + 内容相似度 × 0.5
       + 关键词重叠度 × 0.2
```

### 分组策略
1. 按场景类型分组（INFORMATION_QUERY, SYMPTOM_ANALYSIS等）
2. 组内计算相似度
3. 相似度≥0.6的分为一组
4. 每组≥3个任务则合并

### 规则合并
- 追问阈值: 选择最高风险
- 安全规则: 合并所有规则
- 场景类型: 选择最高风险场景
